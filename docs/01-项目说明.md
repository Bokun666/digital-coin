# 项目说明

## 一、项目名称

DigitalCoin

项目英文名：DigitalCoin  
项目仓库名：digital-coin  
项目类型：加密货币交易辅助、AI 分析、策略生成、风控复盘与因子权重演化系统

---

## 二、项目定位

DigitalCoin 是一个面向个人加密货币投资与交易学习的辅助系统。

它不是自动交易机器人，也不是喊单系统。

系统的核心目标是帮助用户把币圈交易从“凭感觉操作”，逐步升级为：

```text
信息收集
  ↓
AI 分析
  ↓
用户判断
  ↓
策略生成
  ↓
风险检查
  ↓
仓位计算
  ↓
手动执行
  ↓
交易记录
  ↓
复盘总结
  ↓
因子权重优化
  ↓
策略迭代
```

系统不会替用户直接做买卖决策，也不会承诺收益。

它的核心价值是：

1. 帮助用户在交易前想清楚为什么要操作
2. 帮助用户检查风险、仓位、止损、止盈
3. 帮助用户整理市场信息和 AI 分析结果
4. 帮助用户记录自己的判断和实际结果
5. 帮助用户通过复盘优化策略
6. 帮助用户逐步形成自己的交易方法论

---

## 三、项目背景

用户长期接触过加密货币市场，但过去一直以浅尝辄止为主。

用户经历过：

1. 在杭州做 Java 开发时接触加密货币，当时公司老板在做自己的交易所。
2. 疫情期间 ETH 大涨、显卡挖矿火热，用户买显卡组装矿机挖 ETH。
3. 用户挖到 ETH 后基本直接卖出，整体没亏没赚。
4. 用户曾小额参与山寨币交易，整体也是没亏没赚。
5. 2023 年 11 月，用户看到鱼池上线 KAS 矿池，判断 KAS 可能上涨。
6. 用户当时只投入约 200 元，后来涨到约 600 元后卖出，获得约 3 倍收益。
7. 用户认为自己过去不是完全看不到机会，而是没有系统化方法、仓位规划、跟踪机制和复盘体系。
8. 当前用户希望系统性学习币圈交易、投资、网格、合约、链上数据、宏观数据、AI 辅助分析和策略优化。

因此，DigitalCoin 的项目目标不是简单做一个记账软件，而是构建一个个人化的交易辅助与策略进化系统。

---

## 四、项目核心目标

DigitalCoin 的长期目标是帮助用户建立自己的加密货币交易认知系统。

具体目标包括：

1. 建立自选币和机会池
2. 记录每次交易前的想法和判断
3. 系统性检查交易风险
4. 自动计算仓位、止损亏损、盈亏比
5. 使用 GPT 辅助分析市场信息
6. 对比 GPT 判断和用户自己的判断
7. 基于 K 线、宏观、链上、合约等数据生成策略建议
8. 保存每一次策略执行结果
9. 通过复盘判断哪些数据因子有效、哪些无效、哪些误导
10. 根据复盘结果调整因子权重
11. 逐步优化适合用户自己的策略模型

---

## 五、项目不是做什么

DigitalCoin 不是以下系统：

1. 不是自动交易机器人
2. 不是交易所
3. 不是钱包
4. 不是资产托管工具
5. 不是喊单工具
6. 不是量化高频交易系统
7. 不是稳赚策略系统
8. 不是代替用户做决策的系统

MVP 阶段明确不做：

1. 自动下单
2. 自动转账
3. 自动提现
4. 钱包私钥管理
5. 助记词管理
6. 高频交易
7. 自动套利
8. 实盘交易机器人
9. 绕过用户确认的交易执行
10. 带交易权限或提现权限的交易所 API 集成

## 六、项目安全边界

资产安全是本项目最高优先级。

系统必须遵守以下安全边界：

1. 不保存钱包私钥
2. 不保存助记词
3. 不保存交易所 API Secret 到前端
4. 不开启交易所提现权限
5. 不在代码中硬编码任何 API Key
6. 不将 `.env`、`.env.local` 提交到 GitHub
7. 不将真实 API Key 发送给 AI 或 Codex
8. 不让 GPT 直接执行交易
9. 不让 GPT 绕过风险检查模块
10. 不引入来源不明的 npm 包
11. 不无脑批准 pnpm build scripts
12. 不为了功能速度牺牲安全性

---

## 七、系统核心原则

### 1. 用户是最终决策者

GPT、策略模块、因子模型只能提供辅助分析和参数建议。

最终是否操作、投入多少、是否止损、是否止盈，都必须由用户手动确认。

### 2. 先记录，再交易

每一笔交易前，系统都应该要求用户记录：

1. 币种
2. 操作类型
3. 入场理由
4. 当前价格
5. 计划入场价
6. 止损价
7. 止盈价
8. 投入金额
9. 最大可接受亏损
10. 是否使用杠杆
11. 保证金模式
12. 当前情绪状态

如果没有明确计划，不建议交易。

### 3. 先风控，再收益

系统优先关注：

1. 这笔交易最多亏多少钱
2. 亏损是否超过账户总资金可接受范围
3. 是否无止损
4. 是否追涨
5. 是否情绪化交易
6. 是否在重大宏观事件前重仓
7. 是否使用过高杠杆
8. 是否使用全仓合约

收益预期必须排在风险检查之后。

### 4. GPT 是分析员，不是执行员

GPT 可以做：

1. 市场信息总结
2. K 线截图分析
3. 宏观数据解读
4. 合约数据分析
5. 链上数据分析
6. 风险提示
7. 策略参数建议
8. 用户判断对比
9. 复盘辅助

GPT 不能做：

1. 自动下单
2. 自动修改仓位
3. 自动转账
4. 自动提现
5. 直接调用带交易权限的接口
6. 绕过 RiskCheck
7. 绕过 PositionSizing

### 5. 策略必须可复盘

任何策略建议都必须保存：

1. 当时的市场信息
2. 当时的 GPT 分析
3. 用户自己的判断
4. 策略参数
5. 风险评分
6. 仓位计算
7. 用户是否执行
8. 实际交易结果
9. 盈亏情况
10. 复盘结论

没有复盘的数据，不能用于策略优化。

### 6. 因子权重必须可追踪

系统中的每个数据因子都应该有权重。

初始阶段，所有核心因子权重可以设置为 100。

后续根据复盘结果调整：

1. 有效因子提高权重
2. 无效因子降低权重
3. 误导因子明显降低权重
4. 样本不足时不大幅调整
5. 用户违反计划的交易不用于评估因子有效性

所有权重变化都必须保存历史记录。

## 八、系统核心模块总览

DigitalCoin 按功能可以分为五大层：

1. 基础交易辅助层
2. 信息与市场快照层
3. AI 分析层
4. 策略层
5. 因子权重演化层

这五层共同构成系统的完整交易辅助闭环。

---

## 九、第一层：基础交易辅助层

基础交易辅助层是 MVP 阶段必须完成的核心。

### 1. WatchCoin 自选币模块

用于记录用户关注的币种和机会来源。

典型字段：

1. 币种符号
2. 币种名称
3. 所属赛道
4. 关注理由
5. 关键支撑位
6. 关键压力位
7. 风险标签
8. 当前状态
9. 备注

适用场景：

1. BTC / ETH 等主流币跟踪
2. SOL / BNB 等强势币跟踪
3. KAS / ZEC 等矿币机会跟踪
4. Meme 币短线机会记录
5. 新矿池上线机会记录
6. 新交易所上线机会记录
7. 山寨币热点观察

### 2. TradePlan 交易计划模块

用于记录每一次交易前的计划。

它的核心目标是防止用户冲动交易。

典型字段：

1. 币种
2. 操作类型
3. 方向
4. 计划投入金额
5. 当前总资金
6. 当前价格
7. 入场价
8. 止损价
9. 止盈价
10. 杠杆倍数
11. 保证金模式
12. 入场理由
13. 最大可接受亏损
14. 是否有重大宏观事件
15. 是否追涨
16. 是否设置止损
17. 当前情绪状态
18. 计划状态

### 3. RiskCheck 风险检查模块

用于根据交易计划自动生成风险评分。

风险评分因素包括：

1. 是否没有止损
2. 是否使用合约
3. 杠杆是否大于 3 倍
4. 是否使用全仓
5. 是否处于 FOMO、焦虑、想回本、冲动状态
6. 是否临近重大宏观事件
7. 是否追涨
8. 最大亏损是否超过总资金 2%

风险等级：

1. 0 - 30：低风险
2. 31 - 60：中风险
3. 61 - 80：高风险
4. 81 - 100：极高风险

极高风险时，系统应提示：

```text
不建议操作。
该交易可能属于情绪化交易或过度风险交易。
```

### 4. PositionSizing 仓位计算模块

用于计算一笔交易的真实风险。

输入：

1. 总资金
2. 计划投入金额
3. 杠杆倍数
4. 入场价
5. 止损价
6. 止盈价

输出：

1. 实际仓位价值
2. 止损亏损金额
3. 止盈盈利金额
4. 盈亏比
5. 止损亏损占总资金比例
6. 仓位建议

该模块重点解决一个问题：

```text
这笔交易如果失败，最多会亏多少钱？
```

### 5. TradeRecord 交易记录模块

用于记录真实交易结果。

典型字段：

1. 关联交易计划
2. 币种
3. 操作类型
4. 方向
5. 入场时间
6. 出场时间
7. 入场价
8. 出场价
9. 投入金额
10. 杠杆倍数
11. 盈亏金额
12. 盈亏比例
13. 是否遵守计划
14. 出场原因

### 6. Review 交易复盘模块

用于记录交易后的复盘。

复盘重点：

1. 是否遵守交易计划
2. 是否情绪化操作
3. 盈利是否可复制
4. 亏损是否来自策略错误
5. 亏损是否来自执行错误
6. 哪些判断正确
7. 哪些判断错误
8. 下次如何改进

### 7. Dashboard 首页模块

用于展示系统整体状态。

展示内容：

1. 当前总资产
2. 自选币数量
3. 交易计划数量
4. 高风险计划数量
5. 最近交易记录
6. 最近复盘
7. 胜率
8. 盈亏比
9. 总盈亏
10. 最大回撤
11. 最近风险提醒
12. 最常亏损的交易类型

## 十一、第三层：AI 分析层

AI 分析层用于接入 GPT。

核心模块包括：

1. AIAnalysis
2. UserConclusion
3. AnalysisComparison
4. ParameterRecommendation

### AIAnalysis

保存 GPT 对市场信息的分析结果。

包括：

1. 市场总结
2. 偏多、偏空、中性、震荡判断
3. 置信度
4. 关键支撑
5. 关键压力
6. 利多因素
7. 利空因素
8. 风险点
9. 建议行为
10. 策略类型建议
11. 参数建议
12. 不建议交易的原因
13. GPT 原始回复

### UserConclusion

保存用户自己的判断。

包括：

1. 用户方向判断
2. 用户置信度
3. 用户判断理由
4. 用户计划采用的策略
5. 用户计划参数

### AnalysisComparison

对比 GPT 判断和用户判断。

包括：

1. 方向是否一致
2. 风险判断是否一致
3. 策略选择是否一致
4. 参数差异
5. 后续实际结果
6. 更接近真实结果的一方
7. 对比复盘

### ParameterRecommendation

保存参数建议。

例如现货网格参数：

1. 区间下沿
2. 区间上沿
3. 网格数量
4. 每格利润率
5. 投入金额
6. 是否使用杠杆
7. 杠杆上限
8. 止损条件
9. 停止条件

## 十三、第五层：因子权重演化层

因子权重演化层是系统长期进化的核心。

核心模块：

1. Factor
2. FactorSignal
3. StrategyFactor
4. FactorWeightHistory
5. WeightAdjustmentSuggestion
6. MarketRegime

### Factor 因子

因子是一个可分析的信息维度。

例如：

1. K线趋势因子
2. 支撑压力因子
3. 成交量因子
4. 波动率因子
5. 资金费率因子
6. OI 因子
7. 多空比因子
8. 爆仓热力图因子
9. 宏观事件因子
10. 稳定币流动性因子
11. 链上交易所流入因子
12. 机构持仓因子
13. 新闻情绪因子
14. 矿池/交易所上线因子

### FactorSignal 因子信号

每次分析时，每个因子都可以生成一个信号。

信号包括：

1. 方向
2. 强度
3. 置信度
4. 数据质量
5. 加权分数
6. 原因说明

### StrategyFactor 策略因子配置

不同策略使用不同因子和权重。

例如：

现货网格策略更重视：

1. 支撑压力
2. 波动率
3. K线趋势
4. 成交量
5. 宏观事件

合约短线策略更重视：

1. 资金费率
2. OI
3. 爆仓热力图
4. 多空比
5. K线结构

### WeightAdjustmentSuggestion 权重调整建议

系统根据复盘生成权重调整建议。

例如：

1. 资金费率因子本次有效，建议 +5
2. 新闻情绪因子本次误导，建议 -10
3. OI 因子样本不足，建议暂不调整

第一版不自动调整权重。

流程应为：

```text
系统生成建议
  ↓
用户查看原因
  ↓
用户手动确认
  ↓
系统保存权重变化历史
```

## 十五、核心策略方向

### 1. 现货长期配置

适合：

1. BTC
2. ETH

目标：

1. 建立长期仓位
2. 观察牛熊周期
3. 避免频繁交易

### 2. 现货网格

适合：

1. 震荡行情
2. 主流币
3. 不想频繁盯盘的场景

重点参数：

1. 网格上沿
2. 网格下沿
3. 网格数量
4. 每格利润率
5. 投入金额
6. 停止条件

### 3. 合约网格

适合：

1. 有明确趋势判断
2. 低杠杆
3. 小仓位
4. 高风险训练

重点限制：

1. 优先逐仓
2. 低杠杆
3. 控制强平距离
4. 控制保证金
5. 必须提前设置退出条件

### 4. 山寨币机会策略

适合：

1. 小资金参与
2. 信息差机会
3. 热点赛道

重点观察：

1. 交易所上线
2. 矿池上线
3. 主网升级
4. 空投预期
5. 社区热度
6. 解锁风险
7. 流动性风险

### 5. 新矿币机会策略

来源于用户过去 KAS 的经验。

典型逻辑：

```text
大矿池上线
  ↓
矿工关注度提升
  ↓
算力可能增长
  ↓
社区和市场关注度提升
  ↓
币价可能出现阶段性上涨
```

但该逻辑并不保证成功，需要结合：

1. 是否已经提前暴涨
2. 交易所流动性是否足够
3. 矿工抛压是否增加
4. 项目基本面是否支撑
5. 市场整体环境是否配合

## 十七、后续扩展阶段

### Phase 1：手动 MVP

目标：

建立基础交易辅助闭环。

包括：

1. 自选币
2. 交易计划
3. 风险检查
4. 仓位计算
5. 交易记录
6. 复盘

### Phase 2：AI 分析记录

目标：

引入 GPT 分析，但先不做全自动数据采集。

包括：

1. 用户输入市场信息
2. GPT 分析
3. 用户判断
4. AI 与用户判断对比
5. 策略参数建议保存

### Phase 3：公共数据接入

目标：

接入不需要密钥的公共市场数据。

包括：

1. 行情价格
2. 涨跌幅
3. 成交量
4. 宏观日历
5. 稳定币数据
6. 部分公开链上数据

### Phase 4：策略因子演化

目标：

开始记录因子有效性和权重调整建议。

包括：

1. Factor
2. FactorSignal
3. StrategyFactor
4. StrategyRun
5. StrategyReview
6. WeightAdjustmentSuggestion

### Phase 5：只读账户集成

目标：

在安全评估后，接入交易所只读 API。

要求：

1. 只读权限
2. 禁止提现权限
3. 禁止交易权限
4. API Secret 只保存在服务端环境变量
5. 不暴露到前端

### Phase 6：自动交易评估

该阶段不属于当前目标。

只有在满足以下条件后才可以评估：

1. 手动流程稳定
2. 有足够交易样本
3. 风险模型稳定
4. 有模拟盘验证
5. 有单独安全评估
6. 用户明确确认
7. 可随时关闭自动交易

3. 依赖安全
4. 风控逻辑

6. 复盘有效性
7. 策略可解释性

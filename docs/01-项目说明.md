# Digital Coin 项目说明

## 一、项目名称

Digital Coin

## 二、项目定位

Digital Coin 是一个加密货币交易辅助与风控复盘系统。

它不是自动交易系统，也不是喊单系统。

系统的核心目标是帮助用户在每次交易前完成：

- 交易计划记录
- 风险检查
- 仓位计算
- 止盈止损确认
- 情绪状态确认
- 交易记录
- 交易复盘
- 长期统计分析

本系统不负责替用户自动下单，也不承诺任何收益。

## 三、项目背景

用户长期接触过加密货币市场，经历过：

- 杭州 Java 工作阶段接触交易所和加密货币
- 疫情期间参与 ETH 显卡挖矿
- 小额参与山寨币交易
- 2023 年通过鱼池上线 KAS 矿池的信息，判断 KAS 可能上涨，并小额获得约 3 倍收益
- 当前希望系统性学习币圈交易、投资、网格、合约、链上数据和风控方法

过去的问题不是完全看不到机会，而是缺少系统化方法。

本项目的目标是帮助用户从“偶尔发现机会的人”，逐步转变为“能识别机会、验证机会、控制风险、记录过程、持续复盘的人”。

## 四、核心原则

1. 不自动下单
2. 不保存钱包私钥
3. 不保存助记词
4. 不开启交易所提现权限
5. 不提供稳赚信号
6. 不鼓励高杠杆
7. 每次交易前必须有计划
8. 每次交易必须明确最大可接受亏损
9. 合约交易必须记录杠杆、逐仓/全仓、止损价和强平风险
10. 系统重点防止 FOMO、报复性交易、无止损交易和过度杠杆

## 五、第一版 MVP 目标

第一版只做手动录入，不接交易所 API，不做自动交易。

MVP 需要完成以下闭环：

1. 添加自选币
2. 记录关注理由
3. 创建交易计划
4. 系统执行风险评分
5. 系统计算仓位和最大亏损
6. 用户手动去交易所操作
7. 回来记录真实交易结果
8. 写复盘
9. 统计交易表现

## 六、核心模块

### 1. 自选币模块 WatchCoin

用于记录用户关注的币种和机会来源。

例如：

- BTC
- ETH
- SOL
- BNB
- KAS
- ZEC
- 新矿币
- 山寨币
- Meme 币

记录内容包括：

- 币种
- 所属赛道
- 关注理由
- 关键支撑位
- 关键压力位
- 风险标签
- 当前状态
- 备注

### 2. 交易计划模块 TradePlan

用于在交易前记录计划，强制用户想清楚再操作。

记录内容包括：

- 币种
- 操作类型
- 方向
- 计划投入金额
- 当前价格
- 入场价
- 止损价
- 止盈价
- 杠杆倍数
- 保证金模式
- 入场理由
- 最大可接受亏损
- 是否有重大宏观事件
- 是否追涨
- 是否设置止损
- 当前情绪状态

### 3. 风险检查模块 RiskCheck

根据交易计划自动生成风险评分。

系统根据以下因素增加风险分：

- 没有止损
- 使用合约
- 杠杆过高
- 使用全仓
- 情绪状态为 FOMO、冲动、想回本
- 重大宏观事件前开仓
- 追涨
- 最大亏损超过账户总资金的 2%

风险等级分为：

- 低风险
- 中风险
- 高风险
- 极高风险

极高风险时，系统应明确提示“不建议操作”。

### 4. 仓位计算模块 PositionSizing

用于计算一笔交易的真实风险。

输入：

- 总资金
- 计划投入金额
- 杠杆倍数
- 入场价
- 止损价
- 止盈价

输出：

- 实际仓位价值
- 止损亏损金额
- 止盈盈利金额
- 盈亏比
- 止损亏损占总资金比例
- 仓位建议

### 5. 交易记录模块 TradeRecord

用于记录真实交易结果。

记录内容包括：

- 关联的交易计划
- 币种
- 操作类型
- 方向
- 入场时间
- 出场时间
- 入场价
- 出场价
- 投入金额
- 杠杆倍数
- 盈亏金额
- 盈亏比例
- 是否遵守计划
- 出场原因

### 6. 复盘模块 Review

用于总结每次交易的经验。

记录内容包括：

- 这次交易是否遵守计划
- 盈利是否可复制
- 亏损原因是什么
- 是否有情绪化操作
- 下次如何改进
- 本次经验教训

### 7. 首页 Dashboard

用于展示整体状态。

包括：

- 当前总资产
- 最近交易计划
- 最近交易记录
- 当前高风险计划
- 最近复盘
- 胜率
- 总盈亏
- 最大回撤
- 最常亏损的交易类型

## 七、后续扩展方向

第一版完成后，可以逐步扩展：

1. 接入公共行情 API
2. 添加价格提醒
3. 添加宏观事件提醒
4. 添加资金费率提醒
5. 添加持仓资产总览
6. 添加交易统计图表
7. 添加策略模板
8. 添加只读交易所 API
9. 添加链上数据模块
10. 添加网格策略分析模块

自动交易不属于 MVP 范围，未来是否实现需要单独进行安全评估。

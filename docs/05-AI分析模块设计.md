AI 分析模块设计
一、模块定位

AI 分析模块是 DigitalCoin 的核心增强模块。

它负责将市场信息、用户输入、K 线截图、宏观数据、链上数据、合约数据等内容交给 GPT 进行分析，生成结构化结论，并与用户自己的判断进行对比。

AI 分析模块不是自动交易模块。

GPT 的角色是：

分析员、总结员、风险提示员、策略参数建议员。

GPT 不是：

交易执行员
自动下单机器人
账户管理者
资金托管者
收益保证者
二、AI 分析的输入来源

AI 分析模块可以接受多种输入。

1. 用户手动输入

例如：

我认为 BTC 可能突破
我想开 ETH 现货网格
我看到 SOL 放量上涨
我想分析 KAS 是否有新矿币机会
我准备在 CPI 公布前开仓
2. 市场数据 API

后续可接入：

交易所公共行情 API
Coinglass 数据
DeFiLlama 数据
宏观经济日历
稳定币市值数据
链上交易所流入流出
3. 截图输入

用户可以上传截图，例如：

TradingView K线图
Coinglass 爆仓热力图
交易所行情页面
资金费率页面
持仓截图
新闻截图

截图分析需要注意：

截图可能不完整
周期可能不明确
指标可能被截断
价格可能已经变化
截图中可能包含隐私信息
4. 系统内部数据

包括：

自选币信息
历史交易计划
历史交易记录
历史复盘
策略执行记录
因子权重
因子有效性统计
三、AI 分析输出内容

AI 分析结果必须结构化保存，不能只保存一段随意文本。

建议输出字段包括：

字段	说明
summary	市场总结
bias	偏多、偏空、中性、震荡
confidence	置信度，0-100
keySupport	关键支撑
keyResistance	关键压力
riskPoints	风险点
positiveFactors	利多因素
negativeFactors	利空因素
suggestedAction	建议行为
strategyType	建议策略类型
parameterSuggestion	参数建议
noTradeReason	不建议交易的原因
rawResponse	GPT 原始回复
createdAt	分析时间
四、AI 分析类型
1. 行情分析

用于分析某个币当前是否适合操作。

输入：

币种
当前价格
K线信息
成交量
市场环境
合约数据
宏观事件

输出：

当前偏多、偏空还是震荡
是否适合交易
是否适合观望
关键价位
风险提示
2. 网格参数分析

用于给现货网格或合约网格提供参数建议。

输入：

币种
当前价格
近期高低点
支撑压力
波动率
资金费率
宏观事件
用户计划投入金额

输出：

是否适合网格
建议网格类型
区间上沿
区间下沿
网格数量
每格利润率
建议投入比例
是否使用杠杆
杠杆上限
停止条件
3. 合约风险分析

用于分析合约交易是否过于危险。

输入：

杠杆倍数
保证金模式
入场价
止损价
强平价
资金费率
OI
多空比
爆仓热力图

输出：

是否高风险
是否多头拥挤
是否空头拥挤
是否可能插针
是否建议降低杠杆
是否建议取消交易
4. 山寨币机会分析

用于分析山寨币、矿币、Meme 币机会。

输入：

项目名称
币种
赛道
上线信息
交易所信息
矿池信息
社区热度
流动性
是否已暴涨
解锁风险

输出：

机会类型
信息差是否存在
是否已经过热
参与风险
建议仓位
退出条件
5. 复盘辅助分析

用于在交易结束后帮助用户复盘。

输入：

原始交易计划
GPT 当时分析
用户当时判断
实际交易结果
盈亏情况
是否遵守计划
市场后续走势

输出：

判断是否正确
盈利是否可复制
亏损是否来自策略错误
亏损是否来自执行错误
哪些因子有效
哪些因子误导
是否建议调整权重
五、用户判断与 AI 判断对比

系统必须保存用户自己的判断，而不是只保存 GPT 结论。

原因：

用户需要训练自己的判断能力
后续需要比较用户判断和 GPT 判断谁更准确
可以发现用户容易犯的错误
可以发现 GPT 容易误判的场景

对比内容包括：

对比项	说明
方向判断	用户偏多/偏空/震荡，GPT 偏多/偏空/震荡
风险判断	用户认为风险高低，GPT 认为风险高低
策略选择	用户想做什么策略，GPT 建议什么策略
参数差异	用户设置的参数和 GPT 建议参数差异
最终结果	实际结果更接近谁的判断
复盘结论	为什么一致或不一致
六、AI 分析安全规则
1. 禁止发送敏感信息

禁止发送给 GPT：

钱包私钥
助记词
交易所 API Secret
提现密钥
完整认证 Header
身份证信息
银行卡信息
未打码的敏感截图
2. GPT 不能直接执行交易

GPT 输出只能作为分析建议。

禁止：

GPT 自动下单
GPT 自动修改仓位
GPT 自动转账
GPT 自动调用带交易权限的 API
GPT 跳过风险检查直接生成执行指令
3. GPT 输出必须经过系统风控

任何 GPT 给出的策略建议，都必须进入：

TradePlan
RiskCheck
PositionSizing

只有通过风险检查后，用户才能手动决定是否执行。

八、AI 分析流程

推荐流程：

用户发现机会
    ↓
用户记录自己的初步判断
    ↓
系统收集市场信息
    ↓
生成 MarketSnapshot
    ↓
调用 GPT 分析
    ↓
保存 AIAnalysis
    ↓
对比用户判断与 AI 判断
    ↓
生成 TradePlan 或 StrategyRun
    ↓
执行 RiskCheck
    ↓
用户手动决定是否交易
    ↓
交易后复盘
九、AI 分析模块 MVP 设计

第一版不需要直接接入所有 API。

MVP 可以先支持：

用户手动输入市场信息
用户粘贴 K线观察结论
用户上传或描述截图
系统保存 GPT 分析结果
系统保存用户判断
系统保存 AI 与用户判断对比
后续复盘时引用当时的 AI 分析

第一版先做“记录和分析闭环”，不追求全自动数据采集。
